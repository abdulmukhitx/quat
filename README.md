# Quat

Quat превращает интервальные данные АСКУЭ в проверяемые находки для службы
эксплуатации: где возник перерасход, сколько он стоит, что проверить и был ли
получен эффект после исправления.

## Текущий этап

Первый этап проекта — аудит исходных выгрузок. До проверки схемы, единиц,
полноты и временного покрытия моделирование намеренно не запускается.

1. Скопируйте исходные CSV, XLSX или Parquet в `data/raw/`. Файлы в этой папке
   исключены из Git.
2. При необходимости укажите точные имена столбцов в
   `config/data_contract.json` по примеру `config/data_contract.example.json`.
3. Запустите:

```powershell
python scripts/audit_data.py
```

Результаты появятся в:

- `reports/data_audit.json` — полный машиночитаемый профиль;
- `reports/data_audit.md` — короткий отчёт для просмотра человеком.

CSV анализируются потоково стандартной библиотекой Python. Для XLSX и Parquet
нужны дополнительные зависимости:

```powershell
python -m pip install -e ".[data]"
```

Исходные файлы аудит не изменяет.

## Минимальный контракт данных

Для первого рабочего анализа обязательны:

- временная метка интервала;
- идентификатор счётчика, ячейки или объекта;
- активная энергия за интервал в кВт·ч либо накопительное показание;
- понятная часовая зона;
- описание того, является значение энергией (`кВт·ч`) или мощностью (`кВт`).

Желательны активная/реактивная энергия по направлениям, `cos φ`, напряжение,
ток, наименование объекта и иерархия счётчиков.

## Принципы

- raw-данные неизменяемы;
- все расчёты воспроизводимы;
- временная валидация без случайного перемешивания;
- оценённый перерасход не называется подтверждённой экономией;
- причина находки указывается как гипотеза, пока её не подтвердил специалист;
- интеграция с АСКУЭ остаётся только для чтения.
